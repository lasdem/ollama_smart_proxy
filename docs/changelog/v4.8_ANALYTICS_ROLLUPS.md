# v4.8 — Precomputed analytics rollups, histogram dashboard, admin purge

## Added

- Four wide rollup tables (`analytics_hourly_by_model/ip`, `analytics_daily_by_model/ip`) with incremental updates on new requests and on first transition to `completed` or `error`.
- `GET /proxy/analytics/histogram` — time series from rollups (`view=hourly|daily`, `metric=requests|queue_wait|processing|duration|error_rate`).
- `POST /proxy/admin/db/purge` — optional deletion of `request_logs` and/or rollup tables (admin auth).
- Dashboard **Histogram** tab (Chart.js): hourly (7d) vs daily (90d), metric selector, by-model and by-IP charts.
- Background **rollup retention** (`ANALYTICS_HOURLY_RETENTION_DAYS`, `ANALYTICS_DAILY_RETENTION_DAYS`).
- `/proxy/analytics` may serve Home-oriented fields from rollups when `hours >= ANALYTICS_FROM_ROLLUPS_HOURS_THRESHOLD` (default 72).
- Migration **v4** in `scripts/migrate_db.py`: creates rollup schema (via `create_all`) and backfills from `request_logs`.

## Removed

- API fields: `priority_score_distribution`, `model_bunching_detection`, `average_duration_by_model`, `requests_over_time` (unused by web/terminal dashboards).

## Changed

- Slimmer `/proxy/analytics` response and fewer parallel DB queries.
