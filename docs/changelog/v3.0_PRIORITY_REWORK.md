# Version 3.0 - Priority System Rework
**Date**: 2025-12-19 13:57
**Breaking Change**: Complete priority scoring overhaul

## 🎯 What Changed

### Old System (Confusing):
```
LOWER score = HIGHER priority
-200 = Best (same model loaded)
+300 = Worst (large model swap)
Math: -200 + 20 - 10 + 100 = -90 (??? confusing!)
```

### New System (Intuitive):
```
0 = HIGHEST priority (process FIRST)
Higher numbers = LOWER priority (process LATER)
0 = Best (same model loaded)
500 = Worst (large model swap)
Math: 0 + 20 - 10 + 15 = 25 (clear!)
```

---

## 📊 New Scoring System

### Base Costs (0 = best)

| Scenario | Old Score | New Score | Meaning |
|----------|-----------|-----------|---------|
| Model already loaded | -200 | **0** | Process FIRST |
| Can fit in parallel | -50 | **150** | Process soon |
| Small model swap (<50GB) | +100 | **300** | Process later |
| Large model swap (>50GB) | +300 | **500** | Process LAST |

### Modifiers (Additive)

| Factor | Effect | Example |
|--------|--------|---------|
| **IP active requests** | +10 each | 3 active → +30 |
| **Wait time** | -1/second | 30s wait → -30 |
| **Rate limit** | +5/request (max +100) | 20 requests → +100 |

Base: 0 (loaded)

```

```
