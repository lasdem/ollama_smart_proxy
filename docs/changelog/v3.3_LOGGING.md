# Version 3.3 - Structured Logging (2025-12-19)

## Summary
Replaced print() statements with structured logging system supporting JSON and human-readable formats.

## Key Changes

### New Features
1. **Dual-mode logging**:
   - JSON mode (default) - Machine-readable for Loki/Grafana
   - Human mode - Readable with emojis for local debugging

2. **Structured log fields**:
   - All logs include: timestamp, logger, level, event
   - Request logs: request_id, ip, model, priority, queue_depth, etc.
   - HTTP logs: method, path, status, client

3. **Logger separation**:
   - `[proxy]` - Application events
   - `[uvicorn]` - HTTP access logs

4. **Environment configuration**:
   - `LOG_FORMAT=json|human` (default: json)
   - `LOG_LEVEL=DEBUG|INFO|WARNING|ERROR` (default: INFO)

### Files Added
- `src/log_formatter.py` - Custom formatters for JSON and human modes
- `docs/LOGGING.md` - Comprehensive logging documentation
- `test_logging.py` - Logging system test script

### Files Modified
- `src/smart_proxy.py` - Replaced all print() with logger.info/error()
- Version updated to 3.3

### Migration from v3.2
- **Breaking**: Log format changed from plain text to structured
- **Action**: Update log parsers to handle JSON (or use LOG_FORMAT=human)
- **Tests**: Set `LOG_FORMAT=json` before running test_with_analysis.py

## Event Types

| Event | Emoji | Fields |
|-------|-------|--------|
| proxy_startup | 🚀 | host, port, max_parallel, total_vram_gb |
| request_queued | 📨 | request_id, ip, model, queue_depth |
| request_processing | ⚡ | request_id, priority, vram_gb, loaded, wait_seconds |
| request_completed | ✅ | request_id, duration_seconds |
| request_failed | ❌ | request_id, error |
| vram_poll | 🔍 | request_id, model |
| proxy_shutdown | 👋 | - |

## Grafana Query Examples

```logql
# All proxy events
{logger="proxy"}

# Completed requests only
{logger="proxy"} | json | event="request_completed"

# Slow requests (>10s)
{logger="proxy"} | json | event="request_completed" | duration_seconds > 10

# Errors
{logger="proxy"} | json | level="ERROR"

# Specific model
{logger="proxy"} | json | model="gemma3"
```
