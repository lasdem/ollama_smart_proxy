## Overview

Added comprehensive analytics API endpoint and interactive admin dashboard client for monitoring proxy status and historical performance.

## New Features

### 1. Analytics API Endpoint (`/proxy/analytics`)

**Location:** `src/smart_proxy.py`

New admin-protected endpoint that exposes all analytics queries from `AnalyticsRepository`:

```python
@app.get("/proxy/analytics")
async def proxy_analytics(
    request: Request,
    hours: int = 24,
    group_by: str = "model_name",
    limit: int = 10
)
```

**Features:**
- Admin authentication required (via `verify_admin_access()`)
- Configurable time window (hours parameter)
- Flexible grouping (by model_name or hour)
- Returns comprehensive analytics data including:
  - Request counts by model and IP
  - Average duration by model
  - Priority score distribution
  - Error rate analysis
  - Model bunching detection
  - Requests over time

**Example Request:**
```bash
curl -H "X-Admin-Key: YOUR_KEY" \
  "http://localhost:8003/proxy/analytics?hours=48&group_by=model_name&limit=10"
```

**Example Response:**
```json
{
  "time_range": {
    "start": "2026-02-03T10:00:00",
    "end": "2026-02-04T10:00:00",
    "hours": 24
  },
  "request_count_by_model": [...],
  "request_count_by_ip": [...],
  "average_duration_by_model": [...],
  "priority_score_distribution": [...],
  "error_rate_analysis": [...],
  "model_bunching_detection": [...],
  "requests_over_time": [...]
}
```

### 2. Admin Dashboard Client

**Location:** `scripts/admin_dashboard.py`

Interactive terminal-based dashboard for monitoring proxy health, queue status, VRAM usage, and analytics.

**Features:**

#### Display Modes
- **Rich Interactive Mode** (default): Live-updating dashboard with color formatting
- **Basic Text Mode**: Fallback snapshot mode for environments without rich library

#### Monitoring Capabilities
- **Health Status**: System status, queue depth, request counts, pause state
- **VRAM Status**: Total/used/free VRAM, loaded models with sizes
- **Queue Visualization**: Real-time view of queued/processing requests with priorities
- **Historical Analytics**: 
  - Requests by model (with avg duration)
  - Top IPs by request count
  - Error rates and counts
  - Priority score distributions

#### Authentication
- X-Admin-Key header support
- Command-line argument: `--key YOUR_KEY`
- Environment variable: `PROXY_ADMIN_KEY`

#### Configuration
```bash
--url URL          Proxy URL (default: http://localhost:8003)
--key KEY          Admin key for authentication
--refresh SECONDS  Refresh interval (default: 5)
--hours HOURS      Analytics window in hours (default: 24)
--once             Run once and exit (snapshot mode)
```

**Usage Examples:**

Interactive dashboard:
```bash
python scripts/admin_dashboard.py \
  --url http://localhost:8003 \
  --key YOUR_ADMIN_KEY \
  --refresh 5 \
  --hours 24
```

One-time snapshot:
```bash
python scripts/admin_dashboard.py --once
```

With environment variables:
```bash
export PROXY_URL=http://localhost:8003
export PROXY_ADMIN_KEY=your_secret_key
python scripts/admin_dashboard.py
```

## Technical Details

### API Endpoint Security
- Protected with `verify_admin_access()` function
- Supports three authentication methods:
  1. Static IP whitelist (ADMIN_IPS env var)
  2. Dynamic session (via /proxy/auth)
  3. X-Admin-Key header

### Dashboard Architecture
- **Data Sources**: /proxy/health, /proxy/queue, /proxy/vram, /proxy/analytics
- **Dependencies**: requests (required), rich (optional for enhanced display)
- **Layout**: Split-screen design optimized for 1080p terminals
  - Left: Health, VRAM, Queue panels
  - Right: Analytics tables (models, IPs, errors, priorities)
- **Refresh**: Configurable live updates (default: 5 seconds)

### Analytics Time Windows
- Default: 24 hours
- Configurable via `hours` parameter
- Examples: 1 hour, 24 hours, 168 hours (1 week)

## Benefits

### For Admins
- **Visibility**: Real-time and historical view of proxy performance
- **Troubleshooting**: Quick identification of issues (errors, stuck requests, VRAM exhaustion)
- **Capacity Planning**: Understand usage patterns by model and IP
- **Remote Monitoring**: Can run from laptop to monitor remote proxy

### For Operations
- **No UI Required**: Terminal-based, works over SSH
- **Lightweight**: Minimal dependencies (just requests + rich)
- **Flexible**: Interactive live view or snapshot mode for scripts
- **Authenticated**: Secure admin-only access to sensitive data

## Security Considerations

- Analytics endpoint requires admin authentication
- Admin key should be kept secure (not committed to repos)
- Use HTTPS in production for encrypted transmission
- Consider IP whitelisting for additional security
