# v4.7.3 — Parse `thinking` on `/api/generate` (ollama run)

**Date:** 2026-04-03

## Summary

`ollama run` calls **`/api/generate`**, where Ollama puts reasoning in a **top-level `thinking`** field and the answer in **`response`**. The proxy only parsed **`response`**, so the Web UI and DB never saw reasoning for those requests. **`/api/chat`** uses **`message.thinking`** / **`message.content`**; that path was already handled.

`extract_parts_from_ndjson` now emits **`thinking` then `response`** for `api/generate`, matching the chat endpoint order.
