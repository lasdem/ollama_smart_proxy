# v4.3 — Dashboard Improvements

## Summary
Five dashboard improvements: timeout DB logging, Admin tab with controls, live mode performance, and persistent user settings.

## Changes

### 1. Log Timeout to DB with Error Status
- **File:** `src/smart_proxy.py` — `enqueue_request()` TimeoutError handler
- When a request times out, the proxy now logs a row to the database with `status="error"` and `response_text="[Timeout] Request timed out after Ns (was_in_queue=...)"`.
- Also increments `stats["failed_requests"]`.

### 2. Admin Tab on Dashboard
- **Files:** `static/dashboard/index.html`, `app.css`, `app.js`
- New **Admin** tab with four cards:
  - **Authentication** — Admin key input and Set Key button (moved from header).
  - **Live Streaming** — Go Live / Stop Live buttons with WebSocket status.
  - **Queue Control** — Pause Queue, Resume Queue, Clear Stale buttons.
  - **Debug / Testing** — Simulate DB Down, Restore DB buttons.
- Results from admin actions are displayed in a `<pre>` block below the cards.

### 3. Move Live/Auth Controls to Admin Tab
- Header now shows only a read-only auth status indicator (`#authStatus`).
- The admin key input and Set Key button live exclusively in the Admin tab.
- Conversations tab shows a passive WebSocket status indicator instead of Go Live/Stop Live buttons.

### 4. Live Mode Performance (RAF Batching)
- **File:** `static/dashboard/app.js`
- Streaming chunk DOM updates are now batched via `requestAnimationFrame`.
- Multiple chunks arriving in the same frame are coalesced into a single DOM write pass.
- `scrollThreadToBottom()` is called once per frame instead of per chunk.

### 5. Persistent User Settings (localStorage)
- **File:** `static/dashboard/app.js`
- `homeHours`, `convLimit`, and `historyLimit` selections now persist in `localStorage` (keys: `proxy_home_hours`, `proxy_conv_limit`, `proxy_history_limit`).
- Values survive browser restarts (previously used `sessionStorage` for limits, and nothing for hours).
