**Date**: 2025-12-19 13:07

## 🐛 Bugs Fixed

### Bug 1: IP Active Count Always Showed 2
**Problem**: `tracker.add_request()` was called AFTER releasing the queue lock, so next priority calculation saw stale data.

**Fix**: Moved `tracker.add_request()` INSIDE the queue_lock in queue_worker():
```python
async with queue_lock:
    # ... select request ...
    request_queue.pop(idx)
    # ... logging ...
    
    # NEW: Mark as processing BEFORE releasing lock
    tracker.add_request(selected_request.ip, selected_request.model_name)

# Lock released here
asyncio.create_task(process_request(...))
```

**Result**: Each subsequent request now sees incremented ip_active count.

---

### Bug 2: Models Not Bunched Together
**Problem**: `is_model_loaded()` only checked `currently_loaded`, which is updated 1-5s after model starts loading. Parallel requests for same model didn't see it as "loaded".

**Fix**: Added `recently_started_models` set:
```python
class RequestTracker:
    def __init__(self, vram_monitor: VRAMMonitor):
        self.recently_started_models: set = set()  # NEW
        
    def is_model_loaded(self, model_name: str) -> bool:
        normalized = self._normalize_model_name(model_name)
        return (normalized in self.vram_monitor.currently_loaded or 
                normalized in self.recently_started_models)  # NEW
    
    def add_request(self, ip: str, model_name: str):
        normalized = self._normalize_model_name(model_name)
        self.recently_started_models.add(normalized)  # NEW
    
    def remove_request(self, ip: str, model_name: str):
        normalized = self._normalize_model_name(model_name)
        # Keep if still loaded, remove if swapped out
        if normalized not in self.vram_monitor.currently_loaded:
            self.recently_started_models.discard(normalized)
```

**Result**: 
- Request 1 (gemma3): priority +100 (not loaded)
- Request 2 (gemma3): priority **-200** (sees it in recently_started!)
- Request 3-10 (gemma3): all get -200 → **bunched together** ✅

### Bug 3: VRAM History Not Used
**Problem**: History lookup happened AFTER fuzzy matching, so it would find base model before checking if we had actual measured data.

**Fix**: Reordered checks in `get_vram_for_model()`:
```python
def get_vram_for_model(self, model_name: str) -> Optional[int]:
    # 1. Currently loaded (exact match)
    if model_name in self.currently_loaded:
        return self.currently_loaded[model_name].size_vram
    
    # 2. Historical average FIRST ← MOVED UP!
    if model_name in self.vram_history and self.vram_history[model_name]:
        return int(sum(self.vram_history[model_name]) / len(self.vram_history[model_name]))
    
    # 3. Fuzzy match for currently loaded
    # ...
    
    # 4. Fuzzy match for history
    # ...
```

**Result**: After first load, llama3.3 VRAM data (75GB) is remembered and used for priority calculation.
