# v4.7 — Proxy stability after performance regressions

**Date:** 2026-04-03

## Summary

- **Ollama HTTP:** Restored **per-request** `httpx.AsyncClient` with a full **absolute** `target_url` (no shared global client). The response stream and client are closed in **`streaming_body` `finally`** (`r.aclose()` + `client.aclose()`); the error path closes **`r`** (if opened) and **`client`** before re-raising.
- **Stream completion:** `on_stream_done` schedules **`log_request`**, broadcaster completion, stats, and **`_release_active_slot`** in **`asyncio.create_task`** so `tee_stream` does not block the event loop on slow SQLite or locked DB; slot release stays **idempotent** in `_release_active_slot`.
- **Analytics:** **`ANALYTICS_PARALLEL`** — evaluated at **request time** via `_analytics_parallel_enabled()`. Default: **sequential** queries when `DB_TYPE=sqlite`, **parallel** `gather` otherwise; override with `ANALYTICS_PARALLEL=true|false`.
