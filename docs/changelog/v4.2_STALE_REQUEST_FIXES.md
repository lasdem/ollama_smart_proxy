# v4.2 — Stale Request Handling & Analytics Performance

**Date:** 2026-03-20

## Problem

Requests could get permanently stuck in the proxy due to multiple bugs:
- Orphaned queue entries on timeout (item stays in `request_queue` after `asyncio.TimeoutError`)
- No streaming stall detection (httpx stream could hang indefinitely)
- Broadcaster never notified on error paths (dashboard shows phantom "processing" entries)
- `ip_queued` counter drift on timeout (never decremented)
- Analytics queries degrading with growing `request_logs` table (full table scans, no caching)

## Changes

### Bug Fixes (Critical)

1. **Fixed orphaned queue entry on timeout** (`enqueue_request`): The `TimeoutError` and generic `Exception` handlers now scan and remove the request from both `request_queue` AND `active_requests`. Previously only checked `active_requests`, leaving orphaned entries in the queue.

2. **Fixed `ip_queued` counter drift**: Added `RequestTracker.cancel_queued_request(ip)` helper that properly decrements `ip_queued` without touching `active_request_count`. Used by timeout/error handlers in `enqueue_request`.

3. **Guarded `future.set_result/set_exception`**: All calls now check `future.done()` first to avoid `InvalidStateError` when the worker picks up an already-timed-out request.

4. **Added broadcaster cleanup to `process_request` finally block**: `broadcaster.request_completed()` is now always called as a safety net (already idempotent via `.pop(, None)`).

5. **Added broadcaster cleanup to `enqueue_request` error handlers**: Ensures dashboard is notified even when requests fail before reaching `process_request`.

### New Features

6. **Stale request sweeper** (`stale_request_sweeper()`): Background task running every 30s:
   - Removes queue entries with done/expired futures
   - Removes active requests older than `ACTIVE_REQUEST_MAX_DURATION` if model is no longer loaded on Ollama
   - Logs warnings for long-running but legitimate requests (model still loaded)

7. **Streaming stall detection** (`tee_stream` chunk timeout): New `chunk_timeout` parameter wraps each `__anext__()` call with `asyncio.wait_for()`. Raises `TimeoutError` if no chunk arrives within the window, triggering cleanup via the generator's `finally` block.

8. **Log retention task** (`log_retention_task()`): Background task that hourly deletes rows older than `LOG_RETENTION_DAYS` (disabled by default).

9. **Admin force-clear endpoint** (`POST /proxy/clear-stale`): Manually remove all stale requests from queue and active_requests.

### Analytics & DB Performance

10. **Time-filtered `get_requests_over_time`**: Added `start_time`/`end_time` params (default: last 7 days). Previously scanned the entire table with no WHERE clause.

11. **Analytics response caching**: 60s TTL in-memory cache on `/proxy/analytics`. Prevents the 10 aggregation queries from running on every dashboard refresh.

12. **Composite database indexes**: Added `(source_ip, outgoing_conversation_fingerprint)` and `(timestamp_received, model_name)` for the session-chaining hot path and time-ranged analytics.

13. **Fixed leading-wildcard LIKE**: Changed `/proxy/query_db` model filter from `%model%` to `model%` prefix match (supports `*` wildcard for explicit substring search).

14. **Fixed GROUP_CONCAT portability**: `get_error_rate_analysis` now uses `STRING_AGG` on PostgreSQL and `GROUP_CONCAT` on SQLite.

15. **Optimized fallback recovery**: Replaced N×SELECT de-dupe check with bulk `SELECT ... IN (...)` in batches of 500.

## New Configuration

| Variable | Default | Description |
|---|---|---|
| `ACTIVE_REQUEST_MAX_DURATION` | `600` | Seconds before an active request is considered stale |
| `QUEUE_ENTRY_MAX_AGE` | `REQUEST_TIMEOUT + 60` | Seconds before a queue entry is garbage-collected |
| `STREAM_CHUNK_TIMEOUT` | `300` | Seconds to wait for next streaming chunk before timeout |
| `LOG_RETENTION_DAYS` | `0` | Days to keep log entries (0 = keep all) |
| `ANALYTICS_CACHE_TTL` | `60` | Seconds to cache analytics endpoint responses |

## Files Changed

- `src/smart_proxy.py` — Queue cleanup, sweeper, config, future guards, broadcaster safety
- `src/stream_tap.py` — Per-chunk timeout via `chunk_timeout` parameter
- `src/database.py` — Composite indexes, migration, time-filtered analytics, GROUP_CONCAT fix, bulk fallback recovery
- `src/proxy_endpoints.py` — Analytics caching, LIKE fix, clear-stale endpoint
- `tests/test_stale_sweeper.py` — New: 10 unit tests for all new features
