# v3.7 - Tool Calling Support Fix

**Date:** 2026-02-06
**Status:** ✅ Complete (Current: v3.7.2)

## Overview

Fixed "MODEL_NO_TOOLS_USED" and "Unexpected API Response" errors when using the proxy with AI coding assistants (like Kilo Code VSCode addon) that require tool/function calling capabilities.

## Evolution

### v3.7.0 - Parameter Passthrough ✅
**Problem:** Parameters like `tools`, `tool_choice`, `temperature` were being dropped.

**Solution:** Implemented full parameter passthrough - all request body parameters (except explicitly handled ones) are forwarded to LiteLLM.

### v3.7.1 - Response Format Fixes ✅
**Problem:** Tool call responses weren't properly formatted, causing "Unexpected API Response" errors.

**Solution:** Added tool_calls preservation in response formatting, fixed streaming defaults, improved logging.

### v3.7.2 - Simplified Pass-Through ✅
**Problem:** Response reformatting was still causing issues with tool calling responses.

**Root Insight:** We're a PROXY, not a format converter. We should pass responses through unchanged.

**Solution:** 
- **OpenAI endpoints** (`/v1/chat/completions`, `/v1/completions`): Pass LiteLLM response through directly (already in correct format)
- **Ollama endpoints** (`/api/chat`, `/api/generate`): Still convert (Ollama has different format than OpenAI)
- **Logging**: Made extraction safer with try/catch to never fail requests due to logging

### Simplified Response Handling (v3.7.2)

**Location:** `src/smart_proxy.py` - `format_output()` function

```python
def format_output(response, stream: bool, endpoint_type: EndpointType):
    """
    Pass through LiteLLM responses directly for OpenAI endpoints.
    Only convert format for Ollama endpoints (since Ollama has different format).
    """
    
    # --- OPENAI ENDPOINTS: Pass through as-is ---
    if endpoint_type in [EndpointType.OPENAI_CHAT, EndpointType.OPENAI_LEGACY]:
        if stream:
            # Stream OpenAI format directly from LiteLLM
            async def openai_stream_iterator():
                async for chunk in response:
                    yield f"data: {chunk.model_dump_json()}\n\n"
                yield "data: [DONE]\n\n"
            
            return StreamingResponse(openai_stream_iterator(), media_type="text/event-stream")
        else:
            # Return non-streaming OpenAI response directly
            return JSONResponse(content=response.model_dump())
    
    # --- OLLAMA ENDPOINTS: Convert from OpenAI to Ollama format ---
    # ... (conversion logic for Ollama compatibility)
```

### Safe Logging (v3.7.2)

```python
try:
    # Safely extract response for logging
    if hasattr(response, 'choices') and len(response.choices) > 0:
        message = response.choices[0].message
        if hasattr(message, 'tool_calls') and message.tool_calls:
            tool_names = [tc.function.name for tc in message.tool_calls]
            response_text = f"[TOOL_CALLS: {', '.join(tool_names)}]"
        else:
            response_text = message.content
except Exception as log_error:
    # Don't fail request if logging fails
    logger.warning(f"Failed to extract response for logging: {log_error}")
    response_text = "[LOG_EXTRACTION_FAILED]"
```

## Design Philosophy (v3.7.2)

**The proxy should be transparent:**
- For OpenAI endpoints → Pass response through unchanged
- For Ollama endpoints → Convert format (OpenAI ↔ Ollama)
- For logging → Extract safely without affecting requests
- For errors → Let clients see the real error from backend

This ensures maximum compatibility and minimal surprise for clients.
