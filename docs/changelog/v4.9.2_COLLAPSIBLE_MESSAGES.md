# v4.9.2 — Collapsible Conversation Messages

**Date:** 2026-04-09

## Problem

Long multi-turn conversations in the Conversations tab showed all messages fully expanded, making it hard to navigate to the latest exchange. Users had to scroll past long system prompts and earlier turns to reach the current content.

## Changes

### Dashboard CSS (`static/dashboard/app.css`)

- `.thread-msg` gains `position: relative` and extra left padding (`1.75rem`) to accommodate the gutter.
- **`.thread-msg-gutter`** — Absolutely positioned vertical bar on the left edge of each message. Thin gray line (2px) that highlights blue on hover. Cursor pointer to indicate clickability.
- **`.thread-msg.collapsed`** — Hides `.body`, `.thread-thinking`, `.thread-meta`; removes bottom margin from `.role`.
- **`.thread-msg-preview`** — Single-line truncated preview (120 chars max), hidden by default, shown when message is collapsed.
- Cache-busted CSS/JS versions from `?v=6` to `?v=7`.

### Dashboard JS (`static/dashboard/app.js`)

- **`sessionCollapseState`** — Per-session map (`{ sid: { msgKey: collapsed } }`) that tracks which messages are collapsed/expanded. Keys use `request_id + ':user'`, `request_id + ':assistant'`, or `'system'`.
- **`addGutterAndPreview(msgDiv, msgKey, previewText, sid)`** — Helper that inserts a gutter element and a preview element into any message div, with a click handler that toggles the `collapsed` class and updates `sessionCollapseState`.
- **`showSessionThread`** updated:
  - Computes collapse defaults: if the session has no saved state (first open), all messages except the last user+assistant pair are collapsed. System message is always collapsed by default.
  - If saved state exists (re-render from background refresh or re-opening), preserves user's choices for existing messages. New messages that weren't in the saved state default to expanded.
  - Every message (system, user, assistant) gets a gutter and preview via `addGutterAndPreview`.
- **`backToSessions`** does not clear `sessionCollapseState`, so re-opening a session remembers collapse choices.
- **Streaming compatibility**: `flushChunkUpdates` and `finalizeCompletedRequest` modify inner DOM elements only (`.body.streamable`, `.thread-thinking`), leaving the outer `.collapsed` class untouched.

## Behavior

- **First open**: Old messages are collapsed (showing role header + one-line preview). Only the latest user+assistant pair is expanded.
- **Manual toggle**: Clicking the gutter bar toggles collapse/expand. This state is saved per-session.
- **Background refresh / new turns**: Existing messages keep their user-set collapse state. Newly added messages default to expanded.
- **Re-open after navigating away**: Collapse state persists in memory for the session lifetime.
