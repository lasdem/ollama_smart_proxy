# v4.7.1 — WebUI: show thinking while streaming

**Date:** 2026-04-03

## Summary

- **Dashboard:** Assistant rows did not include a `.thread-thinking` block until `thinking_text` existed in the DB or `liveThinkingAccumulated` was non-empty, so the first thinking WebSocket chunks had no DOM target. Added `ensureThinkingBlock()` on thinking chunks. Stopped collapsing the thinking `<details>` when content chunks arrive. `hasActiveStreaming()` now treats thinking-only streams as active so `loadSessions` does not clobber the thread mid-think.
- **`stream_tap`:** `/api/chat` lines can carry both `message.thinking` and `message.content`; extraction order is thinking first, then content, and both are emitted in one pass through `extract_parts_from_ndjson`.

- `static/dashboard/app.js`, `src/stream_tap.py`, `tests/test_stream_tap.py`
