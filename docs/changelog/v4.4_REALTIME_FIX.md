# v4.4 — Realtime Dashboard Fix

## Problem
The dashboard was always seconds behind, even in "live" mode. Streaming text would flash/reset periodically during active generation.

## Root Cause
WebSocket lifecycle events (`request_started`, `request_completed`) triggered `debouncedLoadSessions()` which:
1. Waited 500ms (debounce timer reset on every event)
2. Fetched ALL sessions from the DB via HTTP
3. **Rebuilt the entire conversation thread DOM** via `showSessionThread()`

Since `response_text` is only written to the DB on stream completion, mid-stream rebuilds replaced live-accumulated text with empty/stale DB data — causing visible resets every ~1-3 seconds.

Secondary cause: `_broadcast()` sent WebSocket messages to all connected clients **sequentially** — one slow connection delayed all others.

## Changes

### Client-side (app.js)

- **Guard thread rebuild during active streaming**: `loadSessions()` no longer calls `showSessionThread()` when any request in the current thread has live accumulated text. The thread DOM is preserved during streaming; only the session list sidebar is updated.

- **Targeted completion handler**: On `request_completed`, instead of refetching all sessions and rebuilding the thread, the client:
  1. Updates the DOM in-place (removes streaming indicator, renders final markdown)
  2. Fetches only the single completed request (`/requests/{id}`) for final metadata
  3. Updates the session list in the background

- **Skip redundant `loadSessions()`**: When `request_started` arrives for a session already being viewed, the full DB refetch is skipped (chunks already flow via WebSocket).

- **Reduced debounce**: `DEBOUNCE_MS` lowered from 500ms to 150ms for snappier updates.

- **Tab-switch auto-refresh**: Clicking Home or Conversations tab now auto-refreshes data (fixes "Main tab does not refresh" bug).

### Server-side (live_broadcaster.py)

- **Concurrent broadcast**: `_broadcast()` now uses `asyncio.gather()` to send to all WebSocket clients in parallel instead of sequentially. One slow connection no longer blocks others.
