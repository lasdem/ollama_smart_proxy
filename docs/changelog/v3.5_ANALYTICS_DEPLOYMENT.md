### Migration Strategy

The migration script:
1. Creates `schema_version` table if not exists
2. Checks current version
3. Runs pending migrations in order
4. Records each migration applied
5. Supports targeted version migrations
6. Can backfill from fallback logs

### Docker Architecture

Multi-stage build:
- Builder stage: Installs dependencies in venv
- Runtime stage: Minimal Python image with venv
- Result: Smaller final image, faster builds

## 🔄 Migration Path

For existing deployments:

1. **Stop the proxy**
   ```bash
   # If running directly
   # Stop the process (Ctrl+C or kill)
   
   # If running in Docker (old setup)
   docker stop ollama-smart-proxy
   ```

2. **Pull updates**
   ```bash
   git pull origin main
   ```

3. **Run migrations** (if using existing database)
   ```bash
   ./.conda/bin/python scripts/migrate_db.py migrate
   ./.conda/bin/python scripts/migrate_db.py backfill
   ```

4. **Deploy with Docker**
   ```bash
   cp .env.example .env
   # Edit .env with your settings
   docker-compose up -d
   ```
