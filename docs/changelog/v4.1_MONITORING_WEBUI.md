# v4.1 - Monitoring Web UI and Stream Tap

**Date:** 2026-02-11  
**Status:** Complete

## Overview

Implements TODO 4.1: Admin monitoring Web UI with real-time streaming, finished-request history with full response persistence, ChatGPT-like conversation grouping by session, raw JSON and metadata views, all protected by existing admin authentication.

## New Features

### 1. Stream tap and response persistence

- **Module:** `src/stream_tap.py`
- Response stream from Ollama is **tee'd**: bytes are forwarded unchanged to the client while NDJSON is parsed per endpoint (`/api/chat`, `/api/generate`, `/v1/chat/completions`, `/v1/completions`).
- Accumulated response text is persisted on stream end via `RequestLogRepository.log_request(..., response_text=...)`.
- Client-visible behaviour is unchanged; parsing failures are logged and do not affect the stream.

### 2. Live broadcaster and WebSocket

- **Module:** `src/live_broadcaster.py`
- In-memory broadcaster stores in-flight request content and pushes events to connected admin clients.
- **WebSocket:** `GET /proxy/live` (auth via query `key=PROXY_ADMIN_KEY` or IP/session). Events: `request_started` (metadata includes `session_id`, `prompt_preview` from last message), `chunk`, `request_completed`.
- New subscribers receive current active request_ids and their accumulated text so they can join in-progress streams.

### 3. Session ID and content-based conversation grouping

- **RequestLog:** Columns `session_id` and `outgoing_conversation_fingerprint` (nullable, indexed). A request belongs to the same session as a prior one when its message history (minus the last user message) matches that prior request’s messages plus assistant response; fingerprint hash is used for the match. New sessions get `{source_ip}_{timestamp}`.
- **QueuedRequest:** Optional `session_id` set at enqueue and passed to live `request_started` metadata for the dashboard.
- **Migration:** `scripts/migrate_db.py` migrations v2 (session_id) and v3 (outgoing_conversation_fingerprint).

### 4. Request list and detail API

- **GET /proxy/query_db:** Query parameters include `session_id`, `sort_by`, `sort_order`; response includes `session_id` per request.
- **GET /proxy/requests/{request_id}:** Returns one request with full metadata, `prompt_text`, `response_text` (admin auth required).

### 5. Monitoring dashboard

- **URL:** `/proxy/dashboard` (and `/proxy/dashboard/*` for assets). Admin auth required.
- **Static files:** `static/dashboard/` (index.html, app.css, app.js).
- **Tabs:** **Conversations** (primary): session list, thread view with User · IP · date and Assistant · model · duration; **Go live** / **Stop live** and optional **Auto-refresh (10s)**; live streaming into the open thread (no duplicate rows); auto-open session when it goes live; metadata toggle per message; empty-prompt (warmup) sessions filtered out. **History:** table with filters and detail modal (Request/Response and Raw JSON tabs).
