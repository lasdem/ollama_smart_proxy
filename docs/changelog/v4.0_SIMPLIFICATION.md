# Version 4.0 - Architectural Simplification: Pure HTTP Proxy

**Date**: 2026-02-06  
**Type**: Major architectural change (breaking)

## Overview

Complete removal of LiteLLM dependency and conversion to pure HTTP proxy. This addresses persistent compatibility issues with various Ollama clients (particularly tool calling with Kilo Code VSCode extension) by eliminating all request/response format conversion and manipulation.

## Architectural Philosophy

**Core Insight**: "We're a proxy with smart queueing, not a format converter"

The proxy's value is in intelligent request scheduling based on VRAM availability and fair queueing, not in format translation. By forwarding requests as-is, we:
- Maintain perfect compatibility with all Ollama clients
- Eliminate an entire class of compatibility bugs
- Simplify the codebase significantly
- Reduce dependencies and attack surface

## Major Changes

### Removed
- **LiteLLM dependency** - No longer needed for format conversion
- **`format_output()` function** - ~100 lines of response manipulation logic deleted
- **EndpointType enum** - No longer tracking endpoint types for formatting
- **Response reformatting** - All responses passed through unchanged

### Modified
- **`process_request()`** - Rewritten to forward raw HTTP instead of using LiteLLM's `acompletion()`
  - Now uses `httpx.AsyncClient` to build and forward requests
  - Preserves original headers, query parameters, and body
  - Returns `StreamingResponse` wrapping raw Ollama response stream
  
- **`QueuedRequest` dataclass** - Updated structure:
  - Added: `raw_request: Request` - Store original FastAPI request
  - Added: `path: str` - Endpoint path for routing
  - Removed: `endpoint_type: EndpointType` - No longer needed
  
- **`enqueue_request()` signature** - Changed from:
  ```python
  async def enqueue_request(request: Request, endpoint_type: EndpointType)
  ```
  to:
  ```python
  async def enqueue_request(request: Request, path: str)
  ```

- **Endpoint handlers** - Simplified to pure forwarding:
  ```python
  @app.post("/api/chat")
  async def handle_ollama_chat(request: Request):
      return await enqueue_request(request, "api/chat")
  ```

### Preserved
All core proxy features retained:
- ✅ VRAM-aware priority queue
- ✅ Model affinity scheduling
- ✅ IP-based fairness
- ✅ Wait time starvation prevention
- ✅ Request ID tracking
- ✅ Database logging (PostgreSQL/SQLite)
- ✅ Security (IP whitelist + admin key)
- ✅ Admin dashboard
- ✅ Analytics

## Breaking Changes

### For Users
- **None** - The external API remains identical
- All existing clients should continue working unchanged
- In fact, clients that failed with v3.x should now work correctly

### For Developers/Contributors
- Removing LiteLLM dependency requires re-installation of requirements
- Code that referenced `EndpointType` or `format_output()` will no longer work
- Version bump from 3.x to 4.0 indicates this is a major architectural shift

## Migration Guide

### Updating an Existing Installation

1. Pull latest code:
   ```bash
   git pull origin main
   ```

2. Reinstall dependencies (LiteLLM removed):
   ```bash
   # If using conda
   conda activate ./.conda
   pip install -r requirements.txt
   
   # Or with Docker
   docker-compose build
   ```

3. Restart the proxy - no configuration changes needed

### For Custom Modifications
If you've modified the proxy code:
- Any custom endpoint handlers should be updated to use `enqueue_request(request, "path")` pattern
- Remove any references to `EndpointType` or `format_output()`
- Review changes in `process_request()` - it's now pure HTTP forwarding

## Technical Implementation

### New HTTP Forwarding Logic

```python
async def process_request(request: QueuedRequest):
    """Forward request to Ollama as-is, without any format conversion"""
    base_url = OLLAMA_API_BASE.rstrip("/")
    target_url = f"{base_url}/{request.path.lstrip('/')}"
    
    # Preserve query parameters
    if request.raw_request.url.query:
        target_url += f"?{request.raw_request.url.query}"

    # Forward headers (filter out hop-by-hop headers)
    headers = dict(request.raw_request.headers)
    headers.pop("host", None)
    headers.pop("content-length", None)

    # Build and send request with original body
    client = httpx.AsyncClient(base_url=base_url, timeout=REQUEST_TIMEOUT)
    req = client.build_request(
        request.raw_request.method,
        target_url,
        headers=headers,
        json=request.body  # Use the parsed JSON body
    )
    r = await client.send(req, stream=True)
    
    # Stream response back unchanged
    response = StreamingResponse(
        r.aiter_raw(),
        status_code=r.status_code,
        headers=dict(r.headers),
        background=BackgroundTask(r.aclose)
    )
    
    return response
```

### Why This Works

1. **Zero Manipulation** - Request body and response stream pass through unchanged
2. **Streaming Preserved** - `StreamingResponse` wrapping `r.aiter_raw()` maintains streaming
3. **Headers Preserved** - All response headers forwarded (only hop-by-hop headers filtered on request)
4. **Status Codes Preserved** - Ollama's HTTP status codes passed through unchanged

## Future Implications

This architectural change sets the foundation for:
- Easier maintenance (less code, fewer dependencies)
- Better debuggability (raw HTTP forwarding is easier to trace)
- Potential for extending to other LLM providers (not just Ollama) with minimal changes
- Focus development efforts on improving queueing/scheduling logic rather than format conversion
