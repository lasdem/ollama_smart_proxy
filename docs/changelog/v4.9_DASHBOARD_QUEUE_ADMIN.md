# v4.9 — Dashboard mobile UX, histogram messaging, queue admin

## Summary

- **Home (mobile)**: Grid and card flex rules adjusted so queue and analytics cards no longer stretch and hide content below the fold; queue body scrolls inside a max-height.
- **Histogram**: Banner for missing key, API errors (403/503), and empty rollup series; chart height capped on narrow viewports; Histogram tab always runs `loadHistogram` when selected.
- **Admin API**: `POST /proxy/clear-queue`, `POST /proxy/cancel-request/{request_id}`, `POST /proxy/stop-request/{request_id}` (admin auth).
- **Stop active**: `QueuedRequest` holds upstream `httpx` response/client while streaming; admin stop sets `admin_abort`, closes the stream, logs `[Stopped by administrator]` and completes live broadcast as cancelled.

## UI

- Admin: Clear Queue, Stop active request (request_id field), confirmations where appropriate.
- Home queue table: Stop (processing) / Cancel (queued) when admin key is set.
- Conversations thread: Stop/Cancel on live assistant rows when applicable.
