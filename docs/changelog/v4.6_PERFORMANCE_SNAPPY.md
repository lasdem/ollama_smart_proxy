# v4.6 — Snappy proxy: shared HTTP client, parallel analytics, lean queries

**Date:** 2026-04-03

## Summary

- **Ollama HTTP:** Single shared `httpx.AsyncClient` for all upstream requests (connection reuse, no per-request client `aclose`). Created in app lifespan; relative paths + `base_url`; shutdown closes the client.
- **Enqueue:** Session fingerprint lookup runs in **`asyncio.to_thread`** so it does not block the event loop.
- **Analytics:** `/proxy/analytics` runs **10 independent DB aggregations in parallel** via `asyncio.gather` + `asyncio.to_thread` (lower wall-clock on cache miss).
- **Database pool:** `DB_POOL_SIZE` and `DB_MAX_OVERFLOW` env vars (defaults 10 / 20).
- **Dashboard:** `query_db` calls use **`fields=`** to skip huge columns: home “recent” omits TEXT blobs; conversations list omits **`request_body`** while keeping fields needed for the thread UI.

## Follow-up: WebUI streaming text visible without "Go live"

Chunk WebSocket messages were gated on `liveMode`, which is only set by the **Go live** button. The dashboard auto-connects `/proxy/live` when a key is set, so `liveMode` stayed false and streamed tokens never updated the thread (only "streaming…" from DB state). **Fix:** apply `chunk` messages whenever received; `flushChunkUpdates` still only updates visible rows.
